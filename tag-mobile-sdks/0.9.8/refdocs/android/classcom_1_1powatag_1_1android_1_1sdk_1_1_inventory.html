Detailed Description

A description of the availability of a product variant.

You should first check whether the variant isForSale(), provided it is this should be followed by checking if the variant hasUnlimitedStock(), in which case the quantity and back order eligibility can safely be ignored. If the variant does not have unlimited stock the getQuantity() will return the current stock level. Even if the quantity is zero the product may still be purchased if it isEligibleForBackOrder() though this will usually take longer to deliver than a standard order and the user should be made aware of this before purchasing. This class is immutable and thread-safe.
