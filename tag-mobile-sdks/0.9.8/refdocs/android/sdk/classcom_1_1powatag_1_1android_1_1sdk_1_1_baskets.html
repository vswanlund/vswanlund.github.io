Detailed Description

Represents the users shopping baskets.

A single shopping basket is maintained per-merchant. A basket is considered to exist - and the list of items in the basket valid - as long as the basket is not empty. Once a basket is emptied adding a new variant for the same merchant will result in the creation of a new basket and you must retrieve a reference to the list of items again.

Gets the current set of non-empty baskets for the user.

The iteration order of the set is based on the insertion order of the baskets. The set returned by this method is read-only, but will reflect any underlying changes made to the baskets.

Broadcast sent when a specific basket is changed.

Will not be sent for bulk changes. This will always be proceeded a ACTION_BASKETS_CHANGED broadcast. The intent will contain a EXTRA_MERCHANT parcel identifying the basket that was changed.
