Detailed Description

A Product, such as a T-Shirt. Information such as SKU (stock keeping unit) are stored at the product variant level. Each product has at least one variant (e.g. small T-Shirt). If a product has options it may have more than variant (corresponding to the valid combinations of option choices).

This class is immutable and thread-safe.

List<ProductOptionChoice> com.powatag.android.sdk.Product.getOptionChoices ( @NonNull final List< ProductOption >  chosenOptions)

Gets the list of the remaining choices and options which match the provided filter query.

The order of the choices will preserve any the order of product options. For a product with "color" and "size" choices, filtering for a particular color (e.g. red) will return the possible size options available for that color. Will return an empty list if the product has single variant, if there are no more options to pick (the optionSelections has at least 1 option for every choice) or if the option selections result in a invalid combination.
